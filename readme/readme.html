1- des deux côtés.

Il faut que le client et le serveur connaissent la "définition" de l'objet commun. Cette définition est une interface qui donne la liste des méthodes présentes dans l'objet.

> javac Calculator.java

Le fichier Calculator.class doit être présent sur le client et sur le serveur.

2- du côté du serveur

> javac CalculatorImpl.java

Présente l’implémentation de l’interface. Il s’agit de la classe des objets distants.

> rmic –iiop CalculatorImpl

Attention : Classpath=.;… pour pouvoir trouver les class locales

Cette instruction crée _CalculatorImpl_Tie.class et _Calculator_Stub.class (c'est une class créée du côté du serveur qui doit être placé à côté du client et qui permet au client de se connecter à l'objet distant tel qu’il est défini dans CalculatorImpl).

> start orbd -ORBInitialPort 1050

Lance l’ORB (le registry)

start ouvre simplement une nouvelle invite de commandes.

> javac CalculatorServer.java

> java CalculatorServer

 

Crée un objet de la classe CalculatorImpl et l’inscrit dans le registre de l’ORB.

3- du côté du client

> javac CalculatorClient.java

N'oubliez pas qu'il faut avoir copié Calculator.class du côté du client avant de compiler l'application cliente.

> java CalculatorClient

Après avoir vérifié le bon fonctionnement en localhost , placez le registry, le serveur d’objet distant et le client sur trois PCs distincts, sauvegardez ensuite les paquets échangés à l’aide de WireShark.
